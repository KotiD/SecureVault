Secure Password Access

Passwords stay on your phone. Login stays fast.

SecureVault combines a mobile vault, QR-based browser extension login, and short-lived central sessions without storing your real passwords in the cloud.

Get The App See How It Works

Key Features

Local-First Password Storage

All real credentials are stored in your local encrypted vault on mobile. Master password is never stored in centralized DB.

QR Session Login

Browser extension generates a QR session. Mobile scans it, resolves credential locally, and sends only mapped session payload.

5-Minute Session TTL

Central session data auto-expires after 5 minutes or is deleted immediately after successful login completion.

Authenticator for New Device

New device setup for an existing email requires an in-app authenticator code from a registered device.

Browser Extension Autofill

User types mapped password in extension, extension decodes and autofills website login securely.

Supabase Session Backend

Central DB stores only session/device metadata (`auth_sessions`, `user_devices`, `device_auth_codes`) and never real credentials.

How It Works

  1. User opens target website and clicks SecureVault extension.
  2. Extension creates session and shows QR.
  3. Mobile app scans QR and verifies session + domain.
  4. Mobile fetches credential from local vault and creates mapped password.
  5. User enters mapped password in extension.
  6. Extension decodes, autofills login, and completes session.
  7. Session is deleted immediately or expires after 5 minutes.
